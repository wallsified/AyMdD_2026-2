\section{Limpieza del Dataset}

Primeramente, haremos modificaciones para a los valores de las fechas (nacimiento, fecha de desaparición, fecha de registro) para pasarlos a un valor \texttt{datetime}. Esto nos ayudará a poder discretizar los valores de edad, mes y año de desaparición. En particular, con edad haremos una segunda discretización para trabajar con grupos de edad (0-12, 13-17, 18-30, 31-50 y 50+).

\begin{lstlisting}[language=python]
# Convertir fechas a datetime
datos['FECHA_NACIMIENTO'] = pd.to_datetime(datos['FECHA_NACIMIENTO'], format='%Y-%m-%d', errors='coerce')
datos['FECHA_DESAPARICION'] = pd.to_datetime(datos['FECHA_DESAPARICION'], format='%Y-%m-%d %H:%M:%S', errors='coerce')
datos['FECHA_REGISTRO'] = pd.to_datetime(datos['FECHA_REGISTRO'], format='%Y-%m-%d %H:%M:%S', errors='coerce')

# Discretizar EDAD
datos['EDAD'] = (pd.Timestamp.now() - datos['FECHA_NACIMIENTO']).dt.days / 365.25
datos['GRUPO_EDAD'] = pd.cut(datos['EDAD'], bins=[0, 12, 18, 30, 50, 100],
                              labels=['0-12', '13-17', '18-30', '31-50', '50+'])

# Discretizar FECHA_DESAPARICION
datos['MES_DESAPARICION'] = datos['FECHA_DESAPARICION'].dt.month
datos['ANO_DESAPARICION'] = datos['FECHA_DESAPARICION'].dt.year
\end{lstlisting}

Luego, removimos las columnas originales de donde viene esta información para poder trabajar con lo que ya se tiene discretizado y aquellas que no contienen información que nos sea de utilidad para ocupar los algoritmos.

\begin{lstlisting}[language=python]
columnas_a_remover = [
    'ID_VICTIMA',
    'CVE_ENT',
    'CVE_MUN',
    'FECHA_NACIMIENTO',
    'FECHA_DESAPARICION',
    'FECHA_REGISTRO',
]

datos_limpio = datos.drop(columns=columnas_a_remover, errors='ignore')
datos_limpio = datos_limpio[~(datos_limpio == 'CONFIDENCIAL').all(axis=1)]
\end{lstlisting}

Algo que notamos con una revisión manual del dataset es que existen muchas entradas con un valor 'CONFIDENCIAL'. Decidimos
dejar estos valores para poder llevar a una interpretación de los mismos posteriormente.

Este dataset modificado será usado tanto en nuestra implementación de Apriori como usando la de \texttt{mlxtend}.

\section{Implementación Propia de Apriori}

Para nuestra implementación lo que realizamos fue crear una clase (y posteriormente un objeto) \texttt{Apriori} que incluyera los métodos necesarios para llevar a la ejecución del algortimo (cálculo de lift, de confianza, etc.). En general, usamos \texttt{frozensets} para tener conjuntos inmutables y qué de esta manera fuera más fácil eliminar duplicados, trabajar con subconjuntos iguales sin orden (\{A, B\} siendo lo mismo que \{B, A\}) y que nuestra ejecución fuera más optima a largo plazo.

Decidimos incluir dos cosas extra dentro del mismo objeto Apriori. La primera fue la funcionalidad para crear las reglas de asociación. Esto para que, al momento de leer el código, fuera más directo entender de donde venían dichas reglas (pues se invocaría haciendo \texttt{Apriori.genera\_reglas\_asociacion()}). La segunda fue la capacidad de realizar el cálculo de soporte individual para poder obtener el valor más cercano a usar en el algoritmo (resultando en este caso $0.05$).

Como nota aparte, por nuestras pruebas manuales, decidimos ocupar un valor de confianza mínima de $0.3$ en el algoritmo pues los resultados tenían más sentido a la realidad.

El código de esta implementación se ve puede leer \href{https://github.com/Yoba7-7/AyMdD_2026-2/blob/main/Tarea%203/scripts/apriori.py}{en este archivo en GitHub}.

Mientras que el de la ejecución se ve a continuación:

\begin{lstlisting}[language=python]
apriori_manual = Apriori(soporte_min=0.05, confianza_min=0.3)
apriori_manual.carga_transacciones(datos_limpio, columnas=datos_limpio.columns)
itemsets_manual = apriori_manual.apriori()
reglas_asociacion = apriori_manual.genera_reglas_asociacion()
df_reglas_manual = apriori_manual.obtiene_reglas_dataframe()
# Guardado como .csv para procesamiento posterior
df_reglas_manual.to_csv('datos_apriori_manual.csv', encoding='utf-8')
\end{lstlisting}

Esta ejecución, en promedio, tomó 3.5 minutos en cada vuelta. Por que en nuestro caso, no fue la implementación más optima.

\section{Usando Apriori de \texttt{mlxtend}}

Para poder trabajar con \texttt{mlxtend} debemos hacer algunas modificaciones a una copia de nuestro dataset limpio (el que usamos anteriormente para ocupar nuestra implementación de Apriori). El dataframe que ocupa \texttt{mlxtend.aprori} debe ser un dataframe booleano o \textit{onehot}. Esto es, un dataframe en donde se de convertir cada valor posible de una columna en una columna booleana. Por ejemplo, si tenemos que los valores en la columna 'SEXO' son HOMBRE, MUJER, etc, entonces se debe detener un columna SEXO=HOMBRE con valores True o False, otra con SEXO=MUJER, etc, etc.

El código de esta implementación se ve de la siguiente manera:

\begin{lstlisting}[language=python]
# Para trabajar con algo que entienda el apriori de mlxtend, se debe convertir a un dataframe booleano/onehot.
# Esto es, convertir cada valor en una columna booleana.

datos_para_mlxtend = datos_limpio.copy()

# En lugar de insertar columna por columna, crear una lista de columnas
columnas_onehot = {}

for columna in datos_para_mlxtend.columns:
    valores_unicos = datos_para_mlxtend[columna].dropna().unique()

    for valor in valores_unicos:
        nombre_col = f"{columna}={valor}"
        columnas_onehot[nombre_col] = (datos_para_mlxtend[columna] == valor).astype(bool)

datos_onehot = pd.DataFrame(columnas_onehot)

itemsets_frequentes_mlxtend = apriori(datos_onehot, min_support=0.05, use_colnames=True)
reglas_mlxtend = association_rules(itemsets_frequentes_mlxtend, metric='confidence', min_threshold=0.3)
reglas_mlxtend = reglas_mlxtend[['antecedents', 'consequents', 'support', 'confidence', 'lift']]
reglas_mlxtend.to_csv('datos_mlxtend.csv', encoding='utf-8')
\end{lstlisting}

Esta ejecución en promedio tomó 35 segundos segundos en cada vuelta, lo que lo hace considerablemente más rápido y mejor optimizado que nuestra implementación.
